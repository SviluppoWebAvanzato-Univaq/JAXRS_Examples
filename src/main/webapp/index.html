JAX-RS Sample application

This is a sample web application based on JakartaEE 10 that runs on Apache Tomcat 10.1 with Java 14 or above.

The application is configured to expose RESTful resources through the JAX-RS reference implementation, Jersey (version 3, for Jakarta EE).

JAX-RS JSON serialization is handled by the Jackson JSON Provider (version 2, for Jakarta EE).

The application shows:

  • how to perform the basic JAX-RS environment setup (class org.univaq.swa.framework.base.RESTApp)
  • how to implement resource classes
  • how to implement a simple WebApplicationException (class org.univaq.swa.examples.exceptions.RESTWebApplicationException)
  • how to implement a custom exception mapper (classes org.univaq.swa.examples.exceptions.AppExceptionMapper and org.univaq.swa.examples.exceptions.JacksonExceptionMapper)
  • how to implement Jackson custom serializers and deserializers for Java classes (package org.univaq.swa.framework.jackson).
  • how to implement a filter to handle CORS preflight requests (class org.univaq.swa.framework.security.CORSFilter)
  • how to implement a simple login resource class to issue an access token (class org.univaq.swa.framework.security.AuthenticationRes)
  • how to declare a JAX-RS authentication annotation (class org.univaq.swa.framework.security.Logged)
  • how to implement a filter to handle JAX-RS authentication annotations (class org.univaq.swa.framework.security.AuthLoggedFilter)
  • how to implement a simple JWT token generation and validation process (class org.univaq.swa.framework.security.JWTHelpers)
